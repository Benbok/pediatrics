## 1. Обзор архитектуры

Система реализует **локальную (offline) лицензионную защиту** на основе:

- **RSA-2048** — асимметричная криптография. Приватный ключ только у разработчика. В приложение встроен публичный ключ (только для проверки).
- **Machine Fingerprint** — SHA-256 хеш от `MachineGuid` (реестр Windows) + `hostname`. Лицензия физически привязана к конкретной машине.
- **ASAR архив** — JS код приложения упакован в бинарный `.asar` архив, что затрудняет прямой просмотр/патчинг кода.

### Почему именно RSA-подпись, а не просто хеш?

Хеш без ключа подделывается тривиально — любой пользователь может вычислить «правильный» хеш своих данных и подставить его в файл. RSA-подпись требует **приватный ключ**, который есть только у вас. Без него создать валидный файл лицензии невозможно — даже зная алгоритм и исходный код.

```
Разработчик                              Пользователь (машина X)
─────────────────                        ──────────────────────────────
[private.pem]                            [Fingerprint = SHA256(MachineGuid + hostname)]
     │                                            │
     │  node generate-license.cjs                 │  Отправляет fingerprint разработчику
     │  --fingerprint <ID>                         │
     │  --user "Иванов"                            │
     ▼                                             │
[license.json]  ──────────────────────────────────►│
  payload: base64(JSON)                            ▼
  signature: RSA-SHA256(payload, privateKey)   [Приложение проверяет]
                                               1. verify(payload, signature, publicKey) ✓
                                               2. fingerprint в payload == текущий fingerprint ✓
                                               3. expiresAt > сейчас ✓
                                               → Активировано ✅
```

## 2. Криптографическая схема

| Параметр | Значение |
|---|---|
| Алгоритм подписи | RSA-SHA256 |
| Длина ключа | 2048 бит |
| Формат приватного ключа | PKCS#8 PEM |
| Формат публичного ключа | SPKI PEM |
| Кодировка payload | Base64 (UTF-8 JSON внутри) |
| Кодировка подписи | Base64 |
| Хеш fingerprint | SHA-256, 64 hex символа |

Используется **только встроенный модуль Node.js `crypto`** — никаких сторонних зависимостей.

## 3. Machine Fingerprint

Fingerprint — уникальный идентификатор компьютера, вычисляется как:

```
fingerprint = SHA256( MachineGuid + "::" + hostname.toLowerCase() )
```

### Источник: Windows Registry

`MachineGuid` хранится в реестре Windows по пути:
```
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Cryptography\MachineGuid
```

Это UUID, создаваемый при установке Windows. Он:
- **Стабилен** при перезагрузках и обновлениях Windows
- **Не меняется** при установке/удалении ПО
- **Уникален** для каждой установки Windows
- Читается через консольную команду `reg query` — без внешних npm библиотек

### Дополнительная компонента: Hostname

`os.hostname()` добавлен как второй фактор:
- Защищает от клонирования виртуальной машины с тем же `MachineGuid`
- При смене имени компьютера лицензия потребует переактивации

### Отображение для пользователя

Полный fingerprint (64 hex символа) сложен для передачи. На экране активации показывается **укороченная читаемая версия** (первые 32 символа, разбитые на группы по 8):

```
A3F7C2D1-E9B5408C-7F3A1D9E-B2C84F17
```

При нажатии «Копировать» в буфер копируется **полный 64-символьный fingerprint** — именно его нужно передать для генерации лицензии.

## 4. Структура файла лицензии

Файл `license.json` содержит два поля:

```json
{
  "payload": "eyJmaW5nZXJwcmludCI6IjdhMmYz...",
  "signature": "WdF8oQ3nBzA1TkLPmR..."
}
```

### Payload (декодированный из Base64)

```json
{
  "fingerprint": "7a2f3b9e1c4d...  (64 hex символа SHA-256)",
  "userName": "Иванов Иван Иванович",
  "issuedAt": "2026-03-22T10:00:00.000Z",
  "expiresAt": "2027-12-31T23:59:59.000Z",
  "version": 1
}
```

| Поле | Описание |
|---|---|
| `fingerprint` | SHA-256 хеш машины-получателя |
| `userName` | Имя лицензиата (отображается для справки) |
| `issuedAt` | Дата и время выпуска лицензии (ISO 8601) |
| `expiresAt` | Дата истечения или `null` для бессрочной |
| `version` | Версия формата лицензии (для будущей совместимости) |

### Хранение на машине пользователя

```
%APPDATA%\pediassist\license.json
```

Файл копируется туда при успешном импорте. При запуске приложения он читается оттуда автоматически.

## 6. Процесс активации пользователем

### Шаг 1 — Установка

Пользователь устанавливает PediAssist через NSIS-установщик (`VaxTrack RF Setup x.x.x.exe`).

### Шаг 2 — Первый запуск → экран активации

При первом запуске (и при каждом запуске без лицензии) приложение показывает экран активации вместо логина:

```
┌─────────────────────────────────────────────────────┐
│  🔑  PediAssist — Активация приложения              │
│                                                     │
│  Приложение не активировано                         │
│  Лицензионный файл не найден                       │
│                                                     │
│  Как активировать:                                  │
│  1. Скопируйте ваш Machine ID ниже                  │
│  2. Отправьте его разработчику                      │
│  3. Нажмите «Импорт лицензии» и укажите файл       │
│                                                     │
│  Ваш Machine ID:                                    │
│  A3F7C2D1-E9B5408C-7F3A1D9E-B2C84F17  [📋]        │
│                                                     │
│  [  Импорт лицензии (license.json)  ]               │
└─────────────────────────────────────────────────────┘
```

### Шаг 3 — Передача Machine ID разработчику

Пользователь нажимает кнопку копирования. В буфер копируется полный 64-символьный fingerprint.

### Шаг 4 — Разработчик выпускает лицензию

Вы (разработчик) запускаете:
```bash
node tools/generate-license.cjs \
  --fingerprint 7a2f3b9e1c4d...64hex... \
  --user "Иванов Иван Иванович" \
  --expires 2027-12-31 \
  --key ./keys/private.pem \
  --out ./license.json
```

### Шаг 5 — Пользователь импортирует лицензию

Вы отправляете пользователю `license.json`. Он нажимает «Импорт лицензии», выбирает файл.

Приложение:
1. Проверяет RSA-SHA256 подпись → должна быть действительна
2. Проверяет fingerprint → должен совпадать с текущей машиной
3. Проверяет срок действия → не истёк
4. Копирует файл в `%APPDATA%\pediassist\license.json`
5. Показывает экран логина

### Шаг 6 — Последующие запуски

При каждом запуске приложение автоматически проверяет `license.json`. Экран активации не показывается.

## 7. Инструкция разработчику — выпуск лицензий

### Первичная настройка (один раз)

```bash
# Генерация RSA-2048 ключей
node tools/generate-license.cjs --generate-keys --out-dir ./keys
```

Выводит:
- `keys/private.pem` — ваш секретный ключ
- `keys/public.pem` — публичный ключ (уже встроен в `verify.cjs`)

> ⚠️ **Сохраните `private.pem` в надёжном месте** (например, зашифрованный архив, безопасный USB). Если потеряете — необходимо будет генерировать новую пару и перевыпускать все лицензии.

### Выпуск новой лицензии

```bash
# Бессрочная лицензия
node tools/generate-license.cjs \
  --fingerprint <64-hex-fingerprint> \
  --user "Фамилия Имя Отчество" \
  --key ./keys/private.pem \
  --out ./license-ivanov.json

# Срочная лицензия (до конкретной даты)
node tools/generate-license.cjs \
  --fingerprint <64-hex-fingerprint> \
  --user "Фамилия Имя Отчество" \
  --expires 2027-12-31 \
  --key ./keys/private.pem \
  --out ./license-ivanov.json
```

### Проверка лицензионного файла

```bash
node tools/generate-license.cjs --verify ./license-ivanov.json --pub ./keys/public.pem
```

Вывод при успехе:
```
✅  Подпись RSA-SHA256 действительна
📋  Содержимое лицензии:
    Пользователь:  Иванов Иван Иванович
    Fingerprint:   7a2f3b9e...
    Выдана:        2026-03-22T10:00:00.000Z
    Истекает:      2027-12-31T23:59:59.000Z
✅  Лицензия действительна
```

### Параметры CLI

| Параметр | Описание | Обязателен |
|---|---|---|
| `--fingerprint <hex>` | 64-символьный SHA-256 fingerprint машины | Да |
| `--user <name>` | Имя лицензиата | Рекомендуется |
| `--expires <date>` | Дата истечения YYYY-MM-DD, без этого параметра — бессрочно | Нет |
| `--key <path>` | Путь к private.pem (по умолчанию: `./keys/private.pem`) | Нет |
| `--out <path>` | Путь для создаваемого license.json (по умолчанию: `./license.json`) | Нет |

## 8. Режим разработки (Dev Mode)

Когда приложение запущено через `npm run electron:dev` (т.е. `app.isPackaged === false`), **лицензионная проверка полностью отключена**.

Используется флаг `isDev = !app.isPackaged`:

```js
// В handlers.cjs
if (isDev) {
    return { valid: true, devMode: true, data: { userName: 'Разработчик', expiresAt: null } };
}
```

В собранном (packaged) дистрибутиве `app.isPackaged` всегда `true`, поэтому обход невозможен.

## 9. Важные замечания по безопасности

### Что защищает система

| Угроза | Защита |
|---|---|
| Пользователь А передаёт установщик пользователю Б | ✅ Fingerprint не совпадёт на машине Б — импорт лицензии провалится |
| Пользователь передаёт `license.json` другому | ✅ Fingerprint не совпадёт — проверка провалится |
| Пользователь подделывает `license.json` | ✅ RSA-подпись сломается — приложение отвергнет файл |
| Пользователь пытается патчить JS код | ✅ ASAR архив осложняет прямое редактирование файлов |
| Пользователь клонирует ВМ с тем же `MachineGuid` | ⚠️ Hostname должен отличаться (нет 100% защиты от клонирования ВМ) |

### Что НЕ защищает система

- **Reverse engineering кода** — ASAR архив не является шифрованием. Опытный разработчик может его распаковать. Для более высокого уровня защиты потребовалась бы обфускация (terser/javascript-obfuscator) + ASAR encryption (доступна в платных сборщиках).
- **Клонирование ВМ** — если пользователь запускает в виртуальной машине и делает snapshot, можно откатиться. Это не является целевой угрозой для данного приложения.

### Контроль доступа в базе данных

Независимо от лицензии, в приложении действует многоуровневая аутентификация:
- Пользователи в БД (`users` таблица), пароли хранятся через bcrypt(10 rounds)
- Роли `admin` и `doctor` с разделением прав через `ensureAuthenticated`/`ensureAdmin`
- Только администратор может создавать новых пользователей

## 10. Хранение ключей

### Рекомендуемые места хранения `private.pem`

1. **Зашифрованный архив** (например, 7zip с паролем AES-256) на отдельном носителе
2. **Password Manager** (Bitwarden, KeePass) — хранение содержимого файла как заметки
3. **Закрытый git-репозиторий** с настроленным шифрованием (git-crypt)
4. **Безопасный USB-носитель** в физически защищённом месте

### Что НИКОГДА не делать

- Не добавлять `keys/private.pem` в git (добавлен в `.gitignore`)
- Не хранить `private.pem` рядом с установщиком приложения
- Не отправлять `private.pem` по email или мессенджерам

## 11. Сценарии ошибок

### «Файл лицензии не найден»

**Причина:** `%APPDATA%\pediassist\license.json` не существует  
**Решение:** Пользователь должен импортировать предоставленный файл лицензии

### «Подпись лицензии недействительна»

**Причина:** Файл лицензии был изменён или создан с неправильным ключом  
**Решение:** Выпустить новый файл лицензии правильным приватным ключом

### «Лицензия привязана к другому компьютеру»

**Причина:** Пользователь пытается использовать `license.json` от другой машины  
**Решение:** Попросить пользователя скопировать Machine ID со своего экрана активации и выпустить лицензию для его машины

### «Срок действия лицензии истёк»

**Причина:** Дата в поле `expiresAt` прошла  
**Решение:** Выпустить новую лицензию с обновлённой датой истечения

### «Не удалось получить идентификатор машины»

**Причина:** `reg query` вернул ошибку (нет доступа к реестру, не Windows)  
**Решение:** Убедиться что приложение запускается на Windows с правами пользователя (не нужны права администратора — `HKLM\SOFTWARE\Microsoft\Cryptography` доступен всем)

## 12. ASAR упаковка

В `package.json` включена упаковка в ASAR:

```json
"asar": true,
"asarUnpack": [
  "node_modules/better-sqlite3/**/*",
  "node_modules/.prisma/**/*",
  "node_modules/@prisma/client/**/*"
]
```

**Почему нативные модули вынесены в asarUnpack:**  
`better-sqlite3` содержит `.node` бинарный файл (нативный модуль C++). Node.js не может загрузить нативные модули из ASAR архива — их нужно распаковать на диск. `asarUnpack` делает это автоматически при установке.

**Результат:**
- `resources/app.asar` — большинство кода (зашифрован бинарно)
- `resources/app.asar.unpacked/` — только нативные модули

```
    │               │       │
    │               │       ├─ isDev? → { valid: true }  ──────────────────────────►│
    │               │       │                                                        │
    │               │       ├─ Нет license.json? → { valid: false }                │

    │               ▼                                                               │
    │           licenseValid = true                                                 │
    │               │                                                               │
    │               ▼                                                               │◄─────(от ActivationPage.onActivated)

    │               ├─ Нет сессии? → LoginPage.tsx                                │

```
