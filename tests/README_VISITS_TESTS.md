# Тесты модуля Приемы

## Обзор

Создан комплексный набор unit-тестов для новой функциональности модуля "Приемы", расширенного согласно формам МЗ РФ.

## Структура тестов

### 1. `visit-validator.test.ts` (37 тестов)

Тесты валидаторов для проверки корректности данных приемов:

- ✅ **DiagnosisEntrySchema** - валидация структуры диагнозов
  - Валидные и невалидные коды МКБ
  - Обязательные и опциональные поля
  
- ✅ **VisitSchema** - базовая валидация приемов
  - Минимальные и полные данные
  - JSON строки и объекты для диагнозов
  - Валидация диапазонов показателей жизнедеятельности
  - Проверка уникальности кодов осложнений/сопутствующих
  
- ✅ **Схемы по типам приемов**:
  - `PrimaryVisitSchema` - первичные приемы
  - `FollowupVisitSchema` - повторные приемы
  - `ConsultationVisitSchema` - консультации (требует referringDoctorId)
  - `EmergencyVisitSchema` - экстренные (требует время)
  - `UrgentVisitSchema` - неотложные (требует время)
  
- ✅ **getVisitSchemaByType** - выбор схемы по типу приема

### 2. `visit-service.test.ts` (20 тестов)

Тесты бизнес-логики сервисов:

- ✅ **Нормализация диагнозов** - структура DiagnosisEntry
- ✅ **Определение типа приема** - первичный vs повторный
- ✅ **Генерация номеров талонов** - формат T-YYYY-NNNNNN
- ✅ **Сериализация/десериализация** - JSON строки и объекты
- ✅ **Валидация крайних случаев** - отсутствующие поля, различные форматы
- ✅ **Переходы статусов** - draft → completed
- ✅ **Валидация по типам** - специфичные требования

### 3. `visit-template-service.test.ts` (13 тестов)

Тесты логики шаблонов приемов:

- ✅ **Применение шаблонов** - слияние с существующими данными
- ✅ **Приоритет данных** - существующие данные перезаписывают шаблон
- ✅ **Критичные поля** - childId, doctorId, id не перезаписываются
- ✅ **Валидация JSON** - структура шаблонов
- ✅ **Фильтрация** - по типу приема, публичные/приватные
- ✅ **Сортировка** - по умолчанию, затем по дате
- ✅ **Сложные структуры** - вложенные объекты и массивы

### 4. `visit-utils.test.ts` (24 теста)

Тесты утилит и вспомогательных функций:

- ✅ **Антропометрия**:
  - Расчет ИМТ (BMI)
  - Расчет ППТ (BSA) по формуле Мостеллера
  - Категории ИМТ по возрасту
  - Форматирование значений
  - Валидация диапазонов
  
- ✅ **Возраст**:
  - Расчет в месяцах
  - Поддержка строковых дат
  - Обработка крайних случаев
  
- ✅ **Вспомогательные функции**:
  - Валидация кодов МКБ
  - Нормализация кодов
  - Форматирование дат и времени

```bash

# С покрытием
```

## Примечания

1. Тесты используют Vitest с конфигурацией из `vitest.config.ts`
2. Все тесты независимы и могут запускаться в любом порядке
3. Моки используются только для внешних зависимостей (window.electronAPI)
4. Тесты проверяют поведение, а не реализацию
