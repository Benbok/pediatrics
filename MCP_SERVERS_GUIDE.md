# Руководство по MCP серверам для PediAssist

## 🎯 Цель: Уменьшение затрат токенов и улучшение качества разработки

MCP серверы позволяют AI ассистенту получать информацию напрямую из источников, вместо включения всего контекста в промпт. Это значительно экономит токены.

---

## ✅ Рекомендуемые серверы (уже настроены)

### 1. **context7** - Документация библиотек
**Экономия токенов:** ⭐⭐⭐⭐⭐
- Получает документацию библиотек по запросу
- Не нужно включать документацию в контекст
- Актуальная информация из официальных источников

**Использование:**
- Документация React, TypeScript, Prisma, Electron
- Примеры кода из официальных источников
- API reference без копирования в промпт

---

### 2. **filesystem** - Работа с файлами
**Экономия токенов:** ⭐⭐⭐⭐
- Читает только нужные файлы по запросу
- Не нужно включать весь код проекта в контекст
- Поиск и фильтрация файлов

**Использование:**
- Чтение конкретных файлов вместо всего проекта
- Поиск по коду через grep/ripgrep
- Работа с файлами без включения в промпт

---

### 3. **sqlite** - Прямой доступ к БД
**Экономия токенов:** ⭐⭐⭐⭐⭐
- Выполняет SQL запросы напрямую
- Не нужно читать схему БД и данные в контекст
- Структурированные ответы

**Пакет:** `mcp-sqlite` (не `@modelcontextprotocol/server-sqlite`)

**Использование:**
- Проверка структуры БД через запросы
- Тестирование данных без чтения файлов
- Валидация схемы через SQL

**Пример экономии:**
- ❌ Без MCP: "Прочитай schema.prisma (500 строк) и проверь структуру"
- ✅ С MCP: "Выполни `PRAGMA table_info(children)`" → получает только нужные данные

---

### 4. **memory** - Кэширование контекста
**Экономия токенов:** ⭐⭐⭐⭐
- Хранит важную информацию между сессиями
- Не нужно повторять контекст в каждой сессии
- Долгосрочная память о проекте

**Использование:**
- Сохранение архитектурных решений
- Запоминание паттернов проекта
- Контекст между разными задачами

---

## 📋 Дополнительные серверы (опционально)

### 5. **puppeteer** - Тестирование веб-приложений
**Экономия токенов:** ⭐⭐⭐
- Автоматизация браузера для тестирования
- Не нужно описывать UI в промпте
- Визуальная проверка результатов

**Когда использовать:**
- Тестирование UI компонентов
- Проверка работы приложения в браузере
- Автоматизация рутинных проверок

**Настройка:**
```json
"puppeteer": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-puppeteer"],
  "disabled": false
}
```

---

### 6. **Git сервер** (альтернатива)
**Экономия токенов:** ⭐⭐⭐
- Работа с Git без чтения истории
- Просмотр изменений через команды
- Анализ коммитов

**Проблема:** Официальный пакет `@modelcontextprotocol/server-git` не найден в npm.

**Альтернатива:** Использовать filesystem + терминальные команды через MCP.

## 💡 Стратегии экономии токенов

### 1. Используйте прямые запросы вместо чтения файлов
```
❌ Плохо: "Прочитай весь файл database.cjs (800 строк) и найди ошибку"
✅ Хорошо: "Используй filesystem для поиска функции saveRecord в database.cjs"
```

### 2. Используйте SQL вместо чтения схемы
```
❌ Плохо: "Прочитай schema.prisma и скажи структуру таблицы children"
✅ Хорошо: "Выполни SQL запрос для получения структуры таблицы children"
```

### 3. Используйте context7 для документации
```
❌ Плохо: "Вот документация Prisma (2000 строк)..."
✅ Хорошо: "Используй context7 для получения документации Prisma по upsert"
```

### 4. Используйте memory для повторяющегося контекста
```
❌ Плохо: Каждый раз объяснять архитектуру проекта
✅ Хорошо: Сохранить в memory архитектурные принципы один раз
```

## 📊 Ожидаемая экономия токенов

| Сервер | Экономия на задачу | Частота использования |
|--------|-------------------|----------------------|
| context7 | 500-2000 токенов | Высокая (документация) |
| filesystem | 200-1000 токенов | Очень высокая (чтение файлов) |
| sqlite | 300-1500 токенов | Средняя (работа с БД) |
| memory | 100-500 токенов | Средняя (контекст) |
| puppeteer | 200-800 токенов | Низкая (тестирование) |

**Итого:** Экономия 1100-5800 токенов на задачу в зависимости от сложности.
