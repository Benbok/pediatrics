# VaxTrack RF - Педиатрический Календарь Прививок

Desktop приложение для интеллектуального отслеживания вакцинации детей в соответствии с законодательством РФ (Приказ 1122н). Включает в себя локальный движок расчета графиков (ATS), систему генерации медицинских сертификатов и ИИ-помощника.

## 🏗 Архитектурные принципы

Приложение следует модульной архитектуре с четким разделением ответственности:
*   **Backend (Electron Main)**: SQLite база данных, управление окнами, нативная печать.
*   **Logic (Adapters & Services)**: Расчет графиков вакцинации, правила догоняющей иммунизации (ATS).
*   **Frontend (React Renderer)**: Интерфейс на базе Tailwind CSS, интерактивные календари и графики.

## 📁 Структура проекта

### 🔹 `src/modules/` — Функциональные модули
- **`vaccination/`**: Движок расчета графиков, правила ATS (Automatic Tracking System), адаптеры данных.
- **`printing/`**: Система предпросмотра и генерации PDF. Использует метод выделенного окна для идеального качества документов.
- **`patients/`**: Управление профилями детей, учет антропометрических данных и факторов риска.
- **`dashboard/`**: Сводная аналитика и визуализация статуса иммунизации.

### 🔹 `electron/` — Системная часть (Main Process)
- `main.cjs`: Управление жизненным циклом, IPC-обработчики (включая многоступенчатую печать).
- `database.cjs`: SQLite слой: хранение профилей, медицинских записей и настроек.
- `preload.cjs`: Безопасный интерфейс доступа к системным функциям.

### 🔹 `public/` — Статические ресурсы
- `print-certificate.html`: Изолированный шаблон для генерации PDF сертификатов.

## 🚀 Быстрый старт

1. **Установка**: `npm install`
2. **Конфигурация**: Создайте `.env.local` на основе `.env.example` (добавьте Gemini API Key для ИИ функций).
3. **Запуск**: `npm run dev`

## 🛠 Технологический стек
- **Core**: React 18 + TypeScript + Vite
- **Shell**: Electron (Main/Renderer изоляция)
- **Styling**: Tailwind CSS + Vanilla CSS (для печати)
- **ORM & Database**: Prisma + better-sqlite3
- **AI Integration**: Google Gemini Pro 1.5

## 📦 Особенности сборки и упаковки

При упаковке приложения через `electron-builder` решены следующие критические задачи:

1.  **Prisma в Electron**: 
    - Конфигурация `files` в `package.json` настроена на принудительное копирование скрытой папки `.prisma`, которая игнорируется по умолчанию.
    - В `database.cjs` реализована динамическая установка путей к бинарным движкам (`PRISMA_QUERY_ENGINE_LIBRARY`) в зависимости от режима (dev/prod).
2.  **Относительные пути (Vite)**: 
    - В `vite.config.ts` установлен `base: './'`, что необходимо для корректной работы ассетов при загрузке через протокол `file://`.
3.  **Hash Routing**: 
    - Используется `createHashRouter` вместо `createBrowserRouter`, так как стандартный браузерный роутинг несовместим с файловой системой Electron.
4.  **Авто-инициализация БД**: 
    - Реализован механизм автоматического создания схемы при первом запуске в production (через Raw SQL), так как Prisma CLI недоступен в упакованном виде. Если схема устарела (отсутствуют колонки), приложение автоматически пересоздает таблицы.
5.  **Идеальная печать и PDF**:
    - **Row-Based Layout**: Решена проблема разрыва карточек между страницами в альбомной ориентации. CSS Grid/Columns в Chromium плохо поддерживают `break-inside` внутри сложных контейнеров. Решение — группировка карточек в строки по 4 (`.timeline-row`) с явным `break-inside: avoid !important` на уровне строки.
    - **PDF Parity**: Шаблон `public/print-certificate.html` синхронизирован по логике рендеринга с основным приложением, обеспечивая идентичный вид PDF-файла и печатной версии.
    - **printToPDF Config**: В `main.cjs` включен параметр `preferCSSPageSize: true`, что заставляет Electron уважать CSS-правила `@page { size: A4 landscape; }` при генерации PDF.
