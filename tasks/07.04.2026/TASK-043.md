## 📋 Описание задачи

Добавить первый экран сценария при первом запуске приложения:

1. Пользователь выбирает сценарий: **«Я администратор»** или **«Я пользователь»**.
2. Для администратора открывается уже реализованный поток `FirstRunSetupPage`.
3. Для обычного пользователя открывается экран ожидания доступа от администратора с Machine ID и инструкциями.

## ✅ Реализация

Реализован требуемый UX первого запуска:

1. Экран выбора сценария: «Войти как администратор» / «Войти как пользователь».
2. Ветка администратора:
	- импорт `private.pem`;
	- автоматический bootstrap с дефолтными данными `admin/admin`;
	- генерация лицензии для текущей машины;
	- переход на логин с предзаполненными `admin/admin`.
3. Ветка пользователя:
	- экран ожидания доступа с Machine ID;
	- инструкция обратиться к админу за `license.json` и учетными данными;
	- кнопка повторной проверки доступа.
4. Dev-исключение:
	- в режиме `import.meta.env.DEV` приложение сразу ведёт на страницу логина,
	  с prefill `admin/admin`, без first-run сценариев.
