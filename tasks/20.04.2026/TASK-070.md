## 📋 Описание задачи

В разделе «Настройки → Безопасность и Данные» реализовать функционал импорта данных
из внешней SQLite базы данных (файл `.db`/`.sqlite` такой же архитектуры).

Функционал позволяет:
1. Выбрать внешний файл базы данных через диалог файловой системы.
2. Отобразить список таблиц с количеством записей из внешнего файла.
3. Выбрать нужные таблицы для импорта (все или отдельные).
4. Выбрать стратегию для каждой таблицы: `REPLACE` (заменить) / `MERGE` (пропустить существующие) / `APPEND` (добавить все).
5. Выполнить импорт: перед импортом автоматически создаётся резервная копия.
6. Получить сводку результатов.

### Контекст
Разработчики и врачи наполняют базу данных на одном устройстве (dev) и хотят
перенести часть или все данные в другой экземпляр приложения без потери текущих данных.

### Ожидаемый результат
- Новая UI-секция «Импорт из базы данных» в разделе Безопасность.
- Два IPC-хендлера: `db:import-get-tables`, `db:import-execute`.
- Сервис `src/services/dbImportService.ts`.
- Типы в `src/types.ts`.
