# TASK-067 — Улучшение сервиса Рефайн: двухэтапная коррекция орфографии и пунктуации

**Дата:** 15.04.2026  
**Модуль:** LLM / VisitForm  
**Статус:** ✅ DONE

## Описание

Сервис рефайна полей в форме приёма пациента (VisitFormPage, DiseaseHistorySection) имел проблемы:
1. Отвергал корректный результат при исправлении опечаток (например "Галовная" → "Головная")
2. Блокировал по хрупким regex-проверкам (юниты измерений неправильно матчились)
3. Не справлялся с длинными текстами (токены заканчивались)
4. Модель теряла фокус, пытаясь одновременно исправлять орфографию, пунктуацию и форматировать

**Решение:** Двухэтапный рефайн с разделением задач:
- **Этап 1 (Орфография)**: модель фокусируется только на исправлении написания слов
- **Этап 2 (Пунктуация)**: модель расставляет знаки и заглавные буквы на уже исправленном тексте

## Acceptance Criteria

✅ **Орфография исправляется:**
- "Галовная боль" → "Головная боль"
- "Залогенность носа" → "Заложенность носа"  
- "Темература 38" → "Температура 38"
- Длинные тексты обрабатываются полностью (не обрезаются)

✅ **Пунктуация расставляется:**
- Запятые между слогами жалоб
- Точки в конце предложений

✅ **Числа и даты сохраняются:**
- Дозы медикаментов не меняются
- Температура 38 остаётся как есть

✅ **Нет утечек метаданных:**
- В результате не появляется "Контекст:", "ПОЛЕ:", "Жалобы на момент поступления" и т.п.

## Реализованные изменения

### `electron/modules/llm/refine.constants.cjs`
- Добавлены `SPELLING_STAGE` и `PUNCTUATION_STAGE` с отдельными промптами
- Функции: `buildSpellingMessages()`, `buildPunctuationMessages()`, `buildSpellingGenerationOptions()`, `buildPunctuationGenerationOptions()`
- Контекст поля перемещён в system prompt (не смешивается с текстом)
- Few-shot примеры с типичными медицинскими опечатками

### `electron/modules/llm/handlers.cjs`
- Удалены хрупкие regex-проверки (PROTECTED_TOKEN_PATTERN, UNIT_TOKEN_PATTERN, fuzzy token matching)
- Упрощена `validateRefinementOutput()`: только empty-check и numeric-preservation
- Добавлен `splitIntoChunks()` для батчинга по предложениям (макс 200 символов)
- Реализован двухэтапный pipeline в `llm:refine-field` handler:
  1. Для каждого чанка → Stage 1 (Spelling)
  2. Для каждого исправленного чанка → Stage 2 (Punctuation)
  3. Валидация чисел после каждого этапа

### `src/modules/visits/components/DiseaseHistorySection.tsx`
- Обновлена функция `getRefineHints()` для хинта "Исправлена орфография"
- Проверка: количество слов одинаково, но написание отличается
