## 📋 Описание задачи

Выделить из `vidal-db` препараты бронхолитического профиля, подготовить JSON-файлы по внутренней схеме medications и затем импортировать их в `dev-db` без потери качества педиатрических дозировок.

### Контекст
- В репозитории уже есть набор генераторов `scripts/generate_*_jsons.py`, которые парсят Vidal и создают импортируемые JSON-файлы.
- Для медикаментозного импорта в этом проекте нужно агрегировать записи по `name_ru + atc_code`, а не по `DocumentID`, иначе повторные Vidal-документы перетирают друг друга при импорте.
- Пользователь отдельно запросил показать объём найденных препаратов до фактического заполнения базы.
- Для первой волны важно не смешивать строгие бронхолитики с контроллерами/антиастматическими препаратами, которые не являются собственно бронходилататорами.

### Ожидаемый результат
- Зафиксирован scope бронхолитиков для первой волны импорта.
- Создан скрипт генерации JSON бронхолитиков по образцу существующих Vidal-генераторов.
- Получены JSON-файлы, совместимые с текущей схемой medications.
- Педиатрические дозировки выделены и нормализованы индивидуально по каждому препарату, где Vidal даёт достаточные сигналы.
- После подтверждения пользователя выполнен импорт в `dev-db` и проведена проверка результата.
