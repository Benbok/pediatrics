# TASK-035 — UI-отображение ошибок валидации и переход к секции с ошибкой

**Дата:** 06.04.2026  
**Модуль:** diseases/form  
**Статус:** ✅ DONE

---

## Описание

В форме болезни backend/frontend ошибки валидации могли отображаться как сырой JSON-массив (например, с `path` и `message`). Требовалось:
- показывать такие ошибки в UI-понятном формате;
- давать пользователю быстрый переход в нужный раздел формы по `path` ошибки;
- автоматически прокручивать к первой проблемной секции после неуспешного сохранения.

---

## Реализация

Файл: `src/modules/diseases/DiseaseFormPage.tsx`

1. Добавлен парсинг структурированных ошибок валидации:
- поддержка `error.issues` / `error.errors`;
- поддержка JSON-массива в `error.message`.

2. Добавлена маршрутизация ошибки в раздел формы по `path[0]`:
- `diagnosticPlan` → «План диагностики»;
- `treatmentPlan` → «План лечения»;
- `symptoms` → «Симптомы и клинические признаки»;
- `clinicalRecommendations` → «Рекомендации»;
- `differentialDiagnosis` / `redFlags` → «Дифференциальная диагностика и красные флаги»;
- базовые поля (`nameRu`, `icd10Code`, `description`, ...) → «Основная информация».

3. Разделы формы получили стабильные `id` + `scroll-mt-24` для аккуратного позиционирования при переходе.

4. В блоке ошибок добавлен список карточек с:
- текстом ошибки;
- подписью раздела;
- кнопкой «Перейти» для скролла к соответствующей секции.

5. При сохранении:
- если найдены структурированные ошибки, показывается friendly summary и автопрокрутка к первой секции с ошибкой;
- иначе поведение fallback-ошибки сохранено.
