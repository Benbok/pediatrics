## 📋 Описание задачи

В форме заболевания при редактировании элемента `diagnosticPlan` наблюдается два дефекта:
1. На `blur` поле "Исследование" может заменяться на нерелевантное название из каталога (пример: ввод "Общий анализ крови (развернутый)" -> "Анализ кала").
2. После такой нормализации может очищаться поле "Обоснование" (rationale).

### Контекст

Нормализация названия теста вызывается асинхронно на `blur`, а обновление `diagnosticPlan` происходит через не-функциональный `setFormData` и может перетирать параллельные изменения в соседних полях. Дополнительно blur-резолвер использует fuzzy matching, что может давать ложные совпадения.

### Ожидаемый результат

- Автозамена по blur выполняется только при надежном (точном) совпадении каталога.
- Пользовательское название теста сохраняется, если точного совпадения нет.
- `rationale` не очищается при blur-нормализации названия теста.
