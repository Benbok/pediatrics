# TASK-033 — CRUD-вкладка «Модуль болезни» в настройках для каталога диагностических тестов

**Дата:** 06.04.2026  
**Модуль:** settings/diseases  
**Статус:** ✅ DONE

---

## Проблема

Управление `DiagnosticTestCatalog` находилось в неудобном потоке редактирования формы болезни и не имело отдельной административной поверхности.
Из-за этого канонические названия исследований и их псевдонимы было сложно просматривать, создавать, править и удалять централизованно.

---

## Решение

Функциональность перенесена в `Settings` в новую вкладку «Модуль болезни».

- Добавлен backend CRUD для `DiagnosticTestCatalog`: список, создание, обновление, удаление
- Добавлены IPC handlers и preload API для работы с каталогом диагностических тестов
- На frontend добавлен сервисный слой и UI для поиска, создания, inline-редактирования и удаления записей
- Для новых и редактируемых записей поддержано управление `aliases[]`
- Состояния каталога вакцин и диагностического каталога разделены, чтобы исключить конфликт `catalogSearch`/`catalogError`
