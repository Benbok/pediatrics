# AGENT ORCHESTRATION — PediAssist

> Инструкция для AI-агента. Следовать строго в указанном порядке.

---

## 🚦 Старт каждой сессии (ОБЯЗАТЕЛЬНО)

```
1. Прочитать tasks/TASKS.md
2. Определить активную задачу (статус IN_PROGRESS или PLANNED)
3. Если задачи нет — запросить у разработчика
4. Открыть tasks/DD.MM.YYYY/TASK-NNN.md активной задачи
5. Синхронизировать контекст через context7 / filesystem MCP
6. Определить текущий незавершённый этап
7. Приступить к выполнению
```

**Нельзя** вносить изменения в код без прочтения TASKS.md и открытия файла задачи.

---

## 📥 Получение новой задачи

### Шаг 1 — Зафиксировать в TASKS.md

Определить следующий ID: найти максимальный TASK-NNN в папке `tasks/` и прибавить 1.

Добавить строку в раздел "Активные задачи":

```markdown
| 02.04.2025 | TASK-001 | patients | Экран истории вакцинации | 🔄 IN_PROGRESS | [→](tasks/02.04.2025/TASK-001.md) |
```

### Шаг 2 — Создать папку для даты

```
tasks/
  02.04.2025/     ← создать если нет
```

### Шаг 3 — Создать детальный файл задачи

Скопировать `tasks/_templates/TASK-TEMPLATE.md` → `tasks/DD.MM.YYYY/TASK-NNN.md`

Заполнить все секции:
- Описание и контекст
- Список файлов которые будут затронуты
- Конкретный план по этапам (Backend → Validation → IPC → Types → Service → Frontend → Tests)
- Заполнить Checklist из AI_CODING_GUIDELINES

### Шаг 4 — Синхронизация контекста

```
→ Запросить context7: актуальное состояние затрагиваемых файлов
→ Сверить план с реальным кодом
→ Скорректировать список файлов и этапов если нужно
```

## ⚙️ Выполнение задачи

### После каждого этапа:

1. Обновить статус этапа в `TASK-NNN.md`:
   ```
   ⬜ TODO → 🔄 IN_PROGRESS → ✅ DONE
   ```

2. Дописать запись в журнал:
   ```markdown
   ### 02.04.2025 14:30 — Этап 2 завершён
   - Добавлен метод getVaccinationHistory в patient.service.ts
   - Unit тест прошёл ✓
   ```

3. Выполнить unit тесты для завершённого этапа

4. Синхронизировать context7 перед началом следующего этапа

### Порядок реализации фичи (из AI_CODING_GUIDELINES):

```
1. Prisma schema → Migration
2. Zod schema → validators/
3. IPC handler → database.cjs
4. Types → src/types.ts
5. Service → src/services/
6. Component → src/modules/
7. Tests
```

---

## ✅ Завершение задачи

### Шаг 1 — Финальный отчёт в TASK-NNN.md

Заполнить секцию "Финальный отчёт":
- Дата завершения
- Краткий итог
- Список всех изменённых файлов

### Шаг 2 — Обновить TASKS.md

Переместить строку из "Активных" в "Выполненные", сменить статус на ✅

### Шаг 3 — Обновить README модуля

Открыть `src/modules/{{module}}/README.md` (или создать если нет) и добавить:

```markdown
## Changelog

### 02.04.2025 — TASK-001
- Добавлена вкладка "История вакцинации" в PatientCard
- Новый IPC handler: `db:get-vaccination-history`
- Новый метод сервиса: `patientService.getVaccinationHistory(childId)`
- Кеширование через CacheService (namespace: vaccination_history, TTL: 5m)
```

## 🔍 Диагностика проблем

**Агент не знает с чего начать?**
→ Прочитать TASKS.md → найти IN_PROGRESS задачу → открыть её файл → посмотреть первый незавершённый этап

**Конфликт с существующим кодом?**
→ Синхронизировать context7 → скорректировать план в TASK-NNN.md → записать в журнал

**Задача заблокирована?**
→ Сменить статус на ⏸️ PAUSED → записать причину в журнал → уведомить разработчика

## 📋 Быстрые правила

| Правило | Детали |
|---------|--------|
| Всегда читать TASKS.md | Перед любым действием |
| Один файл задачи — одна задача | Не мешать несколько задач в один файл |
| Журнал после каждого этапа | Фиксировать что сделано и когда |
| Unit тесты обязательны | После каждого завершённого этапа |
| README модуля обновлять | При завершении задачи, не раньше |
| context7 перед каждым этапом | Не опираться на устаревший контекст |
