## 📋 Описание задачи

Улучшить UX шапки страницы `Пациенты`: сохранить существующий поиск по ФИО и добавить фильтр по дате создания карточки пациента.

### Контекст
- Текущая страница `PatientsModule` поддерживает только поиск по ФИО.
- В списке пациентов backend уже сортирует записи по `createdAt`, но frontend типы и UI не используют этот атрибут.
- Пользователь решил не добавлять фильтр по последнему приему в текущий scope, чтобы сохранить изменение компактным.

### Ожидаемый результат
- В шапке страницы `Пациенты` появляется дополнительный фильтр по диапазону дат создания карточки.
- Поиск и фильтры работают совместно, без `useEffect + useState` для производного списка.
- Дата создания карточки доступна в типизированном frontend flow и используется в UX списка.
