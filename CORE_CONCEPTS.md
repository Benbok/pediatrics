# Core Concepts & Feature Implementation

В данном документе описана техническая реализация ключевых функций приложения PediAssist.

## 1. Модульная Архитектура (Feature-based)

Приложение организовано по модульному принципу. Каждый модуль в `src/modules/` отвечает за конкретную бизнес-область и содержит:
- `components/`: UI компоненты специфичные для модуля.
- `services/`: Бизнес-логика и API вызовы.
- `hooks/`: React хуки для управления состоянием модуля.
- `adapters/`: Преобразователи данных между модулями (например, из Vaccination в Printing).

### Workflow добавления новой фичи:
1. **Типизация**: Определение интерфейса в `src/types.ts` или `types.ts` модуля.
2. **Backend**: Добавление SQL-таблиц/методов в `electron/database.cjs` и IPC-обработчика в `main.cjs`.
3. **Preload**: Экспорт метода в `electron/preload.cjs`.
4. **Adapter/Service**: Создание логики обработки данных.
5. **UI**: Реализация React компонента с использованием Tailwind CSS.

## 2. Vaccination Engine (ATS Logic)

Движок автоматического расчета (Automatic Tracking System) — ядро приложения.

### Техническая реализация:
- **Базовый календарь**: Хранится в `src/modules/vaccination/constants/standardSchedule.ts` (согласно Приказу 1122н).
- **Расчет сроков (Rules)**: 
  - Используются "чистые функции" для вычисления даты следующей прививки на основе даты рождения и предыдущих записей.
  - Учитываются минимальные интервалы (например, 1 месяц между дозами Гепатита B).
- **Догоняющая иммунизация (Catch-up)**: Если прививка пропущена, ИИ (Gemini) или локальный алгоритм пересчитывает график, сохраняя необходимые медицинские интервалы.
- **Адаптация под пациента**: Расчет меняется в зависимости от факторов риска (недоношенность, хронические заболевания), указанных в профиле.

## 3. Система Печати и Генерации PDF

Для достижения идеальной верстки медицинских документов используется гибридный подход.

### Процесс (Workflow):
1. **Предпросмотр**: В основном окне открывается модальное окно `PrintPreview`, где пользователь видит документ "как он будет".
2. **Изоляция**: При нажатии "Открыть PDF" данные передаются в `electron/main.cjs`.
3. **Dedicated Window approach**:
   - Electron создает скрытое окно (`BrowserWindow` с `show: false`).
   - Загружается файл `public/print-certificate.html` (абсолютно чистый HTML/CSS без React и UI приложения).
   - Основной процесс Electron через `executeJavaScript` передает данные в это окно.
   - Окно отрисовывает документ и вызывает `printToPDF`.
4. **Результат**: Генерируется чистый PDF без элементов интерфейса, кнопок или лишних фонов.

## 4. Интеллектуальный помощник (AI Gemini)

Интеграция с ИИ для помощи родителям и врачам.

### Реализация:
- **Context Injection**: При запросе к ИИ, сервис `geminiService.ts` автоматически прикрепляет текущие записи ребенка и факторы риска к системному промпту.
- **Strict Guidelines**: ИИ проинструктирован отвечать строго в соответствии с медицинскими протоколами РФ, избегая общих советов.
- **Markdown Rendering**: Ответы ИИ рендерятся с поддержкой форматирования для удобства чтения.

---

## 5. База данных (SQLite)

Локальное хранение данных обеспечивает 100% приватность.

### Структура:
- **Таблица `children`**: ФИО, дата рождения, вес, пол, факторы риска.
- **Таблица `vaccination_records`**: Связка `child_id`, `vaccine_id`, дата выполнения, препарат, серия, номер дозы.
- **Целостность**: Все операции записи в БД выполняются через асинхронные IPC вызовы, гарантируя отзывчивость UI.
