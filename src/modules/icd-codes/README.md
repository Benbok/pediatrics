# Модуль ICD Codes (Справочник МКБ-10)

## 📋 Общее описание

Модуль `icd-codes` отвечает за работу со справочником Международной классификации болезней 10-го пересмотра (МКБ-10) в педиатрической системе. Модуль предоставляет функционал поиска, фильтрации и навигации по кодам МКБ-10, а также интеграцию с модулем заболеваний для быстрого перехода к связанным заболеваниям.

## 🏗️ Архитектура

Модуль следует архитектурным принципам согласно `AI_CODING_GUIDELINES.md`:

```
┌─────────────────────────────────────────────────────────┐
│  Component Layer (*.tsx)                                │
│  - UI только, NO business logic                          │
│  - Обработка пользовательского ввода                    │
│  - Отображение данных                                    │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  Service Layer (*.service.ts)                           │
│  - ВСЯ бизнес-логика                                     │
│  - IPC вызовы к backend                                  │
│  - Обработка ошибок и логирование                        │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  IPC Handler Layer (electron/modules/icd-codes/*.cjs)  │
│  - ensureAuthenticated middleware                        │
│  - Простые операции с данными                            │
│  - Делегирование логики в Service                       │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  Service Layer (Backend) (service.cjs)                 │
│  - Загрузка данных из JSON файла                        │
│  - Индексация кодов (Map структуры)                      │
│  - Поиск и фильтрация                                    │
│  - Пагинация                                             │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  Data Source (JSON файл)                                 │
│  - src/data/мкб.json                                     │
│  - Структурированные данные МКБ-10                       │
└─────────────────────────────────────────────────────────┘
```

### Принципы архитектуры

1. **Разделение ответственности**: Каждый слой имеет четко определенную роль
2. **Централизованное логирование**: Все ошибки через `logger` (исправлено согласно принципам)
3. **Аутентификация**: Все IPC handlers защищены `ensureAuthenticated`
4. **Индексация данных**: Данные индексируются в памяти для быстрого поиска
5. **Пагинация**: Поддержка пагинации для больших объемов данных
6. **Интеграция**: Связь с модулем заболеваний для навигации

### Особенности модуля

- **In-memory индексация**: Данные загружаются один раз и индексируются в памяти
- **Быстрый поиск**: Использование Map структур для O(1) поиска по коду
- **Фильтрация по категориям**: Фильтрация по первой букве кода (A-Z)
- **Интеграция с заболеваниями**: Автоматический поиск связанных заболеваний по коду МКБ
- **Debounced поиск**: Оптимизация поиска через debounce для снижения нагрузки

## 🎯 Основной функционал

### 1. Загрузка данных

#### Инициализация
- **Источник данных**: JSON файл `src/data/мкб.json`
- **Формат данных**: Структурированный JSON с записями и атрибутами
- **Загрузка**: Происходит при первом обращении к сервису
- **Индексация**: Данные индексируются в памяти для быстрого поиска

#### Структура данных
```json
{
  "data": {
    "records": [
      {
        "uid": "...",
        "attributeValues": [
          {
            "attributeUid": "e0aad94c-bf94-4378-b270-7d385f577124",
            "value": "unique_id"
          },
          {
            "attributeUid": "247cf12b-354c-43b7-81fc-569a0b809103",
            "value": "D32.0"
          },
          {
            "attributeUid": "2fc020c0-a043-4227-bdaf-6eb550076113",
            "value": "Название заболевания"
          }
        ]
      }
    ]
  }
}
```

#### Индексация
После загрузки данные индексируются в три структуры:
- `codes`: Массив всех кодов МКБ
- `byCode`: Map<code, IcdCode> - для быстрого поиска по коду
- `byCategory`: Map<category, IcdCode[]> - для фильтрации по категории

### 2. Поиск кодов

#### Поиск по запросу
- **Параметры**: Поиск по коду или названию
- **Алгоритм**: Нечеткий поиск (contains) по коду и названию
- **Сортировка**: Сначала точные совпадения по коду, затем по названию
- **Пагинация**: Поддержка limit и offset
- **Debounce**: Поиск выполняется с задержкой 300ms для оптимизации

#### Поиск по категории
- **Категории**: A-Z (первая буква кода МКБ)
- **Фильтрация**: Показывает все коды в выбранной категории
- **Сортировка**: По коду (алфавитный порядок)
- **Пагинация**: Поддержка limit и offset

#### Получение по точному коду
- **Использование**: Критично для интеграции с PDF парсингом
- **Нормализация**: Код приводится к верхнему регистру и обрезаются пробелы
- **Результат**: Один код МКБ или null

### 3. Фильтрация

#### По категориям
- **Категории**: A-Z (первая буква кода)
- **UI**: Кнопки-фильтры для быстрого переключения
- **Взаимодействие**: При выборе категории очищается поисковый запрос

#### Поисковый запрос
- **Debounce**: 300ms задержка для оптимизации
- **Очистка**: При выборе категории поиск очищается
- **Приоритет**: Поиск имеет приоритет над фильтром категории

### 4. Пагинация

#### Infinite Scroll
- **Механизм**: Кнопка "Загрузить еще" для подгрузки следующих результатов
- **Размер страницы**: 50 элементов по умолчанию
- **Состояние**: Отслеживание текущей страницы и общего количества

#### Параметры пагинации
- `limit`: Количество элементов на странице (по умолчанию 50)
- `offset`: Смещение для пагинации
- `total`: Общее количество найденных результатов

### 5. Интеграция с модулем заболеваний

#### Навигация к заболеванию
- **При клике на код**: Автоматический поиск связанного заболевания
- **Поиск**: По основному коду (`icd10Code`) или дополнительным кодам (`icd10Codes`)
- **Результат**: Переход на страницу заболевания, если найдено
- **Fallback**: Если заболевание не найдено, ничего не происходит (можно улучшить в будущем)

## 🔌 Сервисы

Frontend сервис для работы с кодами МКБ:

```typescript

icdCodeService.getAllCodes(limit, offset)     // Получение всех кодов
icdCodeService.getCategories()               // Получение списка категорий
```

**Особенности сервиса**:
- Централизованное логирование ошибок через `logger` (исправлено)
- Типобезопасность через TypeScript интерфейсы
- Обработка ошибок с пробросом наверх

### `service.cjs` (Backend)

Backend сервис для индексации и поиска:

#### Основные методы
- `load()`: Загрузка и индексация данных из JSON файла

- `getAll(limit, offset)`: Получение всех кодов с пагинацией

**Особенности**:

## 📝 Примечания для разработчиков

1. **Всегда используйте logger**: Не используйте `console.log`, используйте `logger` (исправлено)
2. **Используйте сервисы**: Все операции через `icdCodeService`, не напрямую через IPC
3. **Нормализация кодов**: Всегда нормализуйте коды перед поиском (верхний регистр, обрезка пробелов)
4. **Пагинация**: Используйте пагинацию для больших объемов данных
5. **Debounce поиска**: Используйте debounce для оптимизации поиска
6. **Обработка ошибок**: Всегда обрабатывайте ошибки и показывайте понятные сообщения пользователю

## 🔄 Обновление данных

Для обновления справочника МКБ-10:

1. **Обновите JSON файл**: Замените `src/data/мкб.json` новым файлом
2. **Перезапустите приложение**: Данные загрузятся при следующем обращении к сервису
3. **Проверьте структуру**: Убедитесь, что структура JSON соответствует ожидаемой
4. **Проверьте UIDs**: Убедитесь, что UIDs атрибутов не изменились

### Структура JSON файла

JSON файл должен содержать:
- `data.records`: Массив записей
- Каждая запись должна содержать `attributeValues` с атрибутами:
  - `UNIQUE_ID`: Уникальный идентификатор
  - `CODE`: Код МКБ
  - `NAME`: Название заболевания
  - `PARENT_ID`: ID родительской записи (опционально)
  - `SORT_FIELD`: Поле для сортировки (опционально)

## ⚠️ Известные ограничения

1. **In-memory хранение**: Данные хранятся в памяти, требуют перезагрузки при обновлении
2. **Размер данных**: Большие JSON файлы могут замедлить загрузку приложения
3. **Нет кеширования**: Данные загружаются каждый раз при перезапуске приложения
4. **Нет обновления**: Нет механизма автоматического обновления справочника
5. **Ограниченный поиск**: Поиск работает только по коду и названию, нет полнотекстового поиска
6. **Нет связи с БД**: Данные не хранятся в базе данных, только в JSON файле
