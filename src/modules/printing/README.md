# Модуль Printing (Печать)

## 📋 Общее описание

Модуль `printing` отвечает за печать и экспорт документов в педиатрической системе. Модуль реализует систему шаблонов печати с поддержкой предпросмотра, прямой печати и экспорта в PDF. Модуль использует паттерн Registry для управления шаблонами и предоставляет единый API для работы с различными типами документов.

## 🏗️ Архитектура

Модуль является **frontend-only** модулем и следует архитектурным принципам согласно `AI_CODING_GUIDELINES.md`:

```
┌─────────────────────────────────────────────────────────┐
│  Component Layer (*.tsx)                                │
│  - UI компоненты шаблонов печати                        │
│  - Компоненты предпросмотра                              │
│  - NO business logic                                     │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  Service Layer (PrintService.ts)                        │
│  - ВСЯ бизнес-логика печати                              │
│  - Валидация данных (через type guards)                │
│  - Управление окнами печати                             │
│  - Обработка ошибок и логирование                        │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  Registry Layer (registry.ts)                            │
│  - Централизованное управление шаблонами                 │
│  - Регистрация и получение шаблонов                      │
│  - Валидация доступности                                 │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────┐
│  Browser API / Electron API                              │
│  - window.print() для браузерной печати                  │
│  - electronAPI.print() для Electron печати               │
│  - electronAPI.exportPDF() для экспорта в PDF           │
└─────────────────────────────────────────────────────────┘
```

### Принципы архитектуры

1. **Разделение ответственности**: Каждый слой имеет четко определенную роль
2. **Паттерн Registry**: Централизованное управление шаблонами через реестр
3. **Централизованное логирование**: Все ошибки через `logger` (исправлено согласно принципам)
4. **Валидация данных**: Type guards для проверки данных перед печатью
5. **Изоляция шаблонов**: Шаблоны полностью изолированы и не зависят от других модулей
6. **Расширяемость**: Легкое добавление новых шаблонов через регистрацию

### Особенности модуля

- **Frontend-only**: Модуль работает только на клиенте, не требует backend
- **Нет IPC handlers**: Не использует IPC для работы с базой данных
- **Браузерная печать**: Использует встроенные возможности браузера/Electron для печати
- **Регистрация шаблонов**: Шаблоны регистрируются при инициализации модуля

## 🎯 Основной функционал

### 1. Система шаблонов

Модуль использует паттерн **Registry** для управления шаблонами печати:

#### Регистрация шаблона
Каждый шаблон должен быть зарегистрирован в `templateRegistry`:
- Уникальный `id`
- Название и описание
- React компонент для рендеринга
- Настройки печати по умолчанию
- Функция валидации данных (type guard)

#### Структура шаблона
```typescript
interface PrintTemplate<TData> {
    id: string;
    name: string;
    description: string;
    component: React.ComponentType<PrintTemplateProps<TData>>;
    defaultOptions: PrintOptions;
    validateData: DataValidator<TData>;
    category?: string;
    icon?: ReactNode;
}
```

### 2. Режимы работы

Модуль поддерживает три режима работы:

#### Предпросмотр (`preview`)
- Открывает модальное окно с предпросмотром документа
- Позволяет просмотреть документ перед печатью
- Кнопки для печати и экспорта в PDF
- Настройки отображения (размер, ориентация)

#### Прямая печать (`direct`)
- Сразу открывает диалог печати браузера
- Без предпросмотра
- Быстрая печать для частых операций

#### Экспорт в PDF (`export_pdf`)
- Экспортирует документ в PDF файл
- Использует Electron API для генерации PDF
- Сохранение файла на диск

### 3. Настройки печати

#### Размеры страниц
- `A4`, `A5`, `A6`, `Letter`

#### Ориентация
- `portrait` (книжная)
- `landscape` (альбомная)

#### Отступы
- Настраиваемые отступы в миллиметрах (top, right, bottom, left)
- По умолчанию: 20mm сверху/снизу, 15mm слева/справа

#### Дополнительные настройки
- `showHeader`: Показывать заголовок
- `showFooter`: Показывать подвал
- `scale`: Масштаб (0.5 - 2.0)
- `customClasses`: Пользовательские CSS классы

### 4. Валидация данных

Каждый шаблон имеет функцию валидации данных:

- **Type guard**: Функция проверяет соответствие данных типу шаблона
- **Перед печатью**: Валидация выполняется перед рендерингом
- **Ошибки**: Возвращает понятные сообщения об ошибках

### 5. Стили печати

Модуль автоматически применяет оптимизированные стили для печати:

#### Автоматические стили
- Скрытие UI элементов (кнопки, навигация)
- Оптимизация шрифтов для печати
- Управление разрывами страниц
- Оптимизация таблиц
- Экономия чернил (управление цветами)

#### CSS классы
- `.no-print`: Скрыть элемент при печати
- `.print-only`: Показать только при печати
- `.page-break`: Разрыв страницы после элемента
- `.page-break-before`: Разрыв страницы перед элементом
- `.avoid-break`: Предотвратить разрыв внутри элемента

### 6. Форматирование данных

Модуль предоставляет утилиты для форматирования:

#### Форматирование дат
- `formatDate(date, format)`: Форматирование даты
  - `short`: ДД.ММ.ГГГГ
  - `long`: ДД месяц ГГГГ
  - `full`: «ДД» месяц ГГГГ г.

#### Форматирование имен
- `formatFullName(surname, name, patronymic, format)`: Форматирование ФИО
  - `full`: Полное ФИО
  - `short`: Фамилия И.О.
  - `initials`: Фамилия И. О.

#### Форматирование веса
- `formatWeight(weight)`: Форматирование веса в граммах

#### Расчет возраста
- `calculateAge(birthDate, format)`: Расчет и форматирование возраста

- **Размер**: A4
- **Содержание**:

```typescript

    id: 'my-template',
    name: 'Мой шаблон',
    description: 'Описание шаблона',
```

## ⚠️ Известные ограничения

1. **PDF экспорт**: Реализован только в Electron (через `window.electronAPI.exportPDF` / IPC в main process)
2. **Браузерная печать**: Зависит от возможностей браузера/Electron
3. **Стили печати**: Некоторые браузеры могут игнорировать некоторые CSS свойства
4. **Размеры страниц**: Размеры рассчитываются для 96 DPI, могут отличаться на других принтерах

## 🎨 Важно про CSS при прямой печати (`invokeBrowserPrint`)

`invokeBrowserPrint` рендерит шаблон **статически** в HTML строку (через `renderToStaticMarkup`) и открывает новое окно печати.

Это означает, что CSS, подключённый через `import './x.css'` внутри React-компонента шаблона, **не попадает** в новое окно автоматически.

Чтобы стили применялись в прямой печати, используйте механизм инлайна:
- добавьте в шаблон поле `styles?: string | string[]` (CSS строкой), и сервис вставит его в `<style>` окна печати.

## ✅ Проверка невалидных данных

Во всех методах `print()` / `preview()` / `exportToPDF()` сервис сначала валидирует данные через `template.validateData`.
При невалидных данных метод должен вернуть `{ success: false, error: '...' }` (без проброса необработанного исключения наверх).
